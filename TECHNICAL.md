# 📘 التوثيق التقني الكامل - نظام إدارة المستشفيات

## 🏗️ التقنيات المستخدمة ولماذا

| التقنية              | الاستخدام          | لماذا اخترناها؟                                        |
| :------------------- | :----------------- | :----------------------------------------------------- |
| **TypeScript**       | لغة البرمجة        | أمان الأنواع (Type Safety)، اكتشاف الأخطاء قبل التشغيل |
| **Node.js**          | بيئة التشغيل       | أداء عالي، غير متزامن، مجتمع ضخم                       |
| **Express.js**       | إطار العمل         | بسيط، مرن، الأكثر استخداماً لبناء APIs                 |
| **PostgreSQL**       | قاعدة البيانات     | قاعدة بيانات علائقية قوية، تدعم العلاقات المعقدة       |
| **Prisma**           | ORM                | أمان الأنواع، تسهيل الاستعلامات، Migration تلقائي      |
| **JWT**              | المصادقة           | تخويل آمن بدون جلسات، مثالي لـ APIs                    |
| **bcrypt**           | تشفير كلمات المرور | خوارزمية قوية مع Salt للحماية من هجمات القوة العمياء   |
| **Zod**              | التحقق من البيانات | يمنع إدخال بيانات خاطئة إلى API                        |
| **Helmet**           | أمان HTTP Headers  | يحمي من هجمات شائعة (XSS، Clickjacking)                |
| **CORS**             | التحكم في الوصول   | يسمح فقط للواجهة الأمامية بالاتصال بالـ API            |
| **Jest + Supertest** | الاختبارات الآلية  | يضمن استقرار النظام بعد أي تعديل                       |

## 📁 شرح كل ملف في المشروع

### 📂 المجلد الرئيسي

| الملف                  | الوظيفة                                        |
| :--------------------- | :--------------------------------------------- |
| `package.json`         | تعريف المشروع، الاعتماديات، سكريبتات التشغيل   |
| `tsconfig.json`        | إعدادات مترجم TypeScript                       |
| `.env`                 | المتغيرات البيئية (قاعدة البيانات، JWT Secret) |
| `prisma/schema.prisma` | تعريف جداول قاعدة البيانات والعلاقات           |
| `backup.ps1`           | سكريبت النسخ الاحتياطي اليومي                  |
| `README.md`            | توثيق المستخدم (تثبيت وتشغيل)                  |
| `LICENSE.txt`          | عقد الترخيص التجاري                            |
| `BACKUP.md`            | سياسة النسخ الاحتياطي                          |
| `TECHNICAL.md`         | هذا الملف - التوثيق التقني                     |

### 📂 `src/` - الكود المصدري

#### 📄 `src/index.ts`

**مدخل التطبيق.** يقوم بـ:

- تحميل المتغيرات البيئية.
- إنشاء تطبيق Express.
- تفعيل Middleware (CORS، Helmet، JSON parser).
- تسجيل المسارات (Routes).
- تشغيل السيرفر.

#### 📂 `src/controllers/`

**منطق الأعمال (Business Logic).** كل ملف يتعامل مع جزء من النظام.

| الملف                         | المسؤول عن                                                |
| :---------------------------- | :-------------------------------------------------------- |
| `auth.controller.ts`          | تسجيل الدخول، تسجيل مستخدم جديد، حذف مستخدم، الملف الشخصي |
| `patient.controller.ts`       | CRUD المرضى                                               |
| `doctor.controller.ts`        | CRUD الأطباء                                              |
| `appointment.controller.ts`   | CRUD المواعيد                                             |
| `medicalRecord.controller.ts` | CRUD السجلات الطبية                                       |
| `prescription.controller.ts`  | CRUD الوصفات الطبية                                       |
| `invoice.controller.ts`       | CRUD الفواتير والمدفوعات                                  |

#### 📂 `src/routes/`

**تعريف نقاط النهاية (API Endpoints).** كل ملف يربط المسارات بالـ Controllers.

| الملف                     | المسارات                 |
| :------------------------ | :----------------------- |
| `auth.routes.ts`          | `/api/auth/*`            |
| `patient.routes.ts`       | `/api/patients/*`        |
| `doctor.routes.ts`        | `/api/doctors/*`         |
| `appointment.routes.ts`   | `/api/appointments/*`    |
| `medicalRecord.routes.ts` | `/api/medical-records/*` |
| `prescription.routes.ts`  | `/api/prescriptions/*`   |
| `invoice.routes.ts`       | `/api/invoices/*`        |

#### 📂 `src/middleware/`

**وظائف وسيطة** تنفذ قبل وصول الطلب إلى Controller.

| الملف                      | الوظيفة                                                         |
| :------------------------- | :-------------------------------------------------------------- |
| `auth.middleware.ts`       | `authenticate`: التحقق من JWT. `authorize`: التحقق من الصلاحيات |
| `validation.middleware.ts` | التحقق من صحة البيانات باستخدام Zod                             |

#### 📂 `src/schemas/`

**مخططات Zod** للتحقق من صحة البيانات المدخلة.

| الملف                   | الوظيفة                              |
| :---------------------- | :----------------------------------- |
| `validation.schemas.ts` | تعريف شكل البيانات المطلوبة لكل مسار |

#### 📂 `src/utils/`

**دوال مساعدة.**

| الملف       | الوظيفة                                                  |

## 🐛 المشاكل المحتملة وحلولها

### 1. `Error: P1001 - Can't reach database server`

**السبب:** قاعدة البيانات غير متصلة أو `DATABASE_URL` خاطئ.
**الحل:**

- تأكد من تشغيل PostgreSQL.
- تحقق من صحة `DATABASE_URL` في `.env`.

### 2. `Error: P3005 - The database schema is not in sync`

**السبب:** تم تعديل `schema.prisma` ولكن Migration لم يتم تطبيقه.
**الحل:**

```bash
npx prisma migrate deploy
npx prisma generate
```
